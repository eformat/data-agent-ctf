# Secrets Management

All CTF secrets are stored in `secrets.enc.yaml`, encrypted with [sops](https://github.com/getsops/sops) + [age](https://github.com/FiloSottile/age). ArgoCD decrypts them at sync time via the sops-age CMP plugin.

## Quick Start

### 1. Install tools

```bash
# sops
brew install sops          # macOS
dnf install sops           # Fedora

# age
brew install age           # macOS
dnf install age            # Fedora
```

### 2. Generate an age key

```bash
age-keygen -o age-key.txt
```

This creates a file with your private key and prints the public key. Copy the public key (starts with `age1...`).

### 3. Configure sops

The repo already has `.sops.yaml` at the root:

```yaml
creation_rules:
  - path_regex: \.enc\.yaml$
    age: >-
      age1YOUR_PUBLIC_KEY_HERE
```

Update the `age:` field with your public key from step 2.

### 4. Create your secrets file

```bash
cp applications/secrets/secrets-example.yaml applications/secrets/secrets.enc.yaml
```

Edit `secrets.enc.yaml` and replace all `CHANGE_ME` values with real credentials.

### 5. Encrypt

```bash
SOPS_AGE_KEY_FILE=age-key.txt sops -e --age $(grep "public key" age-key.txt | awk '{print $NF}') \
  applications/secrets/secrets.enc.yaml > /tmp/encrypted.yaml \
  && mv /tmp/encrypted.yaml applications/secrets/secrets.enc.yaml
```

Or use the in-place editor (opens `$EDITOR`):

```bash
SOPS_AGE_KEY_FILE=age-key.txt sops applications/secrets/secrets.enc.yaml
```

### 6. Verify

```bash
# Should show encrypted values (ENC[AES256_GCM,...])
cat applications/secrets/secrets.enc.yaml

# Should show plaintext
SOPS_AGE_KEY_FILE=age-key.txt sops -d applications/secrets/secrets.enc.yaml
```

## Common Operations

### Decrypt to view/edit

```bash
SOPS_AGE_KEY_FILE=age-key.txt sops -d applications/secrets/secrets.enc.yaml
```

### Edit a single value

```bash
# Decrypt → edit → re-encrypt
SOPS_AGE_KEY_FILE=age-key.txt sops -d applications/secrets/secrets.enc.yaml > /tmp/secrets-dec.enc.yaml
# ... edit /tmp/secrets-dec.enc.yaml ...
SOPS_AGE_KEY_FILE=age-key.txt sops -e --age YOUR_AGE_PUBLIC_KEY /tmp/secrets-dec.enc.yaml > applications/secrets/secrets.enc.yaml
rm /tmp/secrets-dec.enc.yaml
```

Note: the temp file must have `.enc.yaml` extension to match the `.sops.yaml` creation rules.

### Bootstrap ArgoCD decryption

The bootstrap script (`scripts/bootstrap.sh`) creates the sops-age CMP plugin ConfigMap and patches the ArgoCD repo-server to use it. The age private key must be provided as a Secret:

```bash
oc create secret generic sops-age-key \
  --from-file=keys.txt=age-key.txt \
  -n openshift-gitops
```

## Secret Inventory

| Secret | Used By | Keys |
|--------|---------|------|
| `minio-credentials` | MinIO | `MINIO_ROOT_USER`, `MINIO_ROOT_PASSWORD` |
| `trino-credentials` | Trino catalogs | `S3_ACCESS_KEY`, `S3_SECRET_KEY` |
| `spicedb-config` | SpiceDB operator | `preshared_key`, `datastore_uri` |
| `spicedb-token` | MCP servers | `SPICEDB_TOKEN` |
| `keycloak-postgres-credentials` | Keycloak DB | `username`, `password` |
| `keycloak-admin-secret` | Realm setup job | `KEYCLOAK_ADMIN`, `KEYCLOAK_ADMIN_PASSWORD`, `CTF_USER_PASSWORD` |
| `sandbox-secrets` | Sandbox deploy job | `OPENAI_API_KEY` |
| `authbridge-keycloak-admin` | MCP client-registration | `KEYCLOAK_ADMIN_USERNAME`, `KEYCLOAK_ADMIN_PASSWORD` |

## Security Notes

- **Never commit `age-key.txt`** — it's in `.gitignore`
- **Never commit unencrypted `secrets.enc.yaml`** — always encrypt before pushing
- The `secrets-example.yaml` file contains only dummy `CHANGE_ME` values and is safe to commit
- `authbridge-keycloak-admin.KEYCLOAK_ADMIN_PASSWORD` must match the Keycloak initial-admin password (auto-generated by RHBK, check `keycloak-initial-admin` secret)
